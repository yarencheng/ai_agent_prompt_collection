# Hugo Git & Contribution Conventions

Follow these conventions for commits and pull requests:

## Commit Messages
* **Format**: `prefix: Subject` (e.g., `tpl: Add custom index function`).
* **Prefix**: Use package name, common functional topic, `docs:`, or `all:`. No "chore:".
* **Subject**: Imperative mood, starts with capital letter, no trailing period.
* **References**: End body with `See #1234` or `Fixes #1234`.
* **Squashing**: Always squash commits into a single commit before PR.

## Pull Requests
* **Sign CLA**: Required for all contributions.
* **AI Disclosure**: Disclose if substantial parts are autogenerated with AI (e.g., "This PR was written primarily by Claude Code").
* **Narrow Scope**: AI-assisted contributions from non-maintainers should have a narrow scope (e.g., bug fix).
* **Validation**: Ensure `go fmt` is run and `./check.sh` succeeds.
